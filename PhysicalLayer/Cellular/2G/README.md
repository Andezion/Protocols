# 2G - GSM / CDMA (IS-95)

2G - второе поколение сотовых сетей, появившееся в начале 1990-х. Главное отличие от 1G: переход с *аналоговой* передачи на **цифровую**. Речь оцифровывается, сжимается специальным кодером-декодером (CODEC), передаётся в виде битового потока и защищается шифрованием. Два конкурирующих стандарта: **GSM** (Global System for Mobile) - доминировал в Европе и Азии - и **IS-95 (cdmaOne)** - был распространён в США. Позже GSM расширился пакетными надстройками GPRS и EDGE, которые принято называть "2.5G" и "2.75G".

---

## Краткий обзор

| Параметр | GSM | IS-95 (cdmaOne) |
|---|---|---|
| Мультидоступ | TDMA + FDMA | CDMA |
| Модуляция | GMSK | QPSK / BPSK |
| Полоса канала | 200 kHz | 1.25 MHz |
| Скорость речи | 13 kbps (FR), 5.6 kbps (HR) | ~9.6 kbps |
| Частоты | 900 / 1800 / 1900 MHz | 850 / 1900 MHz |
| Данные | GPRS до 114 kbps, EDGE до 384 kbps | до 14.4 kbps |

## Оцифровка речи: от звука к битам

В отличие от *1G*, где голос передавался напрямую как *аналоговый сигнал*, в *2G* голос сначала **оцифровывается и сжимается**.

### Дискретизация (Sampling)

По теореме Найквиста-Шеннона, чтобы восстановить аналоговый сигнал с максимальной частотой $f_{max}$, его нужно дискретизировать с частотой $f_s \geq 2 f_{max}$. Для речи $f_{max} \approx 4$ kHz, поэтому:

$$f_s = 8000 \text{ Гц (выборок в секунду)}$$

Каждая выборка квантуется в 8 бит -> исходный поток = $8000 \times 8 = 64$ kbps (это стандартный поток G.711, как в проводном телефоне).

### Речевое кодирование (CODEC)

64 kbps - слишком много для радиоканала. GSM применяет **линейно-предсказывающий кодер** (LPC - Linear Predictive Coding) с долгосрочным предсказанием (LTP) и возбуждением по регулярной кодовой книге. Этот тип кодека называется **RPE-LTP** (Regular Pulse Excitation - Long-Term Prediction):

- *Полная скорость* (FR, Full Rate): **13 kbps**
- *Половинная скорость* (HR, Half Rate): **5.6 kbps**
- *Усиленная полная скорость* (EFR, Enhanced Full Rate): **12.2 kbps**
- *AMR* (Adaptive Multi-Rate, появился позже): переключается от **4.75 до 12.2 kbps**

Идея LPC: вместо передачи самого сигнала передаются **коэффициенты фильтра** (модель голосового тракта) + короткий вектор ошибки. На приёмнике фильтр синтезирует речь заново.

## Архитектура GSM-сети

```
UE (телефон)
    |
   Um (радиоинтерфейс)
    |
  BTS (Base Transceiver Station - базовая приёмопередающая станция)
    |
   Abis
    |
  BSC (Base Station Controller - контроллер базовых станций)
    |
    A
    |
  MSC (Mobile Switching Centre - центр мобильной коммутации)
    |  \
   HLR  VLR
```

- **BTS** - физическая антенна+радиооборудование, обслуживает одну соту.
- **BSC** - управляет несколькими BTS: распределяет частоты, управляет хэндовером (handover - переключение между базовыми станциями при движении абонента) между своими BTS.
- **MSC** - коммутационный узел, соединяет звонки внутри GSM и с PSTN (Public Switched Telephone Network - телефонная сеть общего пользования).
- **HLR** (Home Location Register - реестр домашних абонентов) - база данных всех абонентов оператора: профиль, услуги, текущее местоположение.
- **VLR** (Visitor Location Register - реестр гостевых абонентов) - временная копия данных об абонентах, находящихся в зоне данного MSC. Ускоряет обработку вызовов без постоянных запросов в HLR.

## TDMA и структура радиоканала GSM

GSM комбинирует **FDMA** (частотное разделение) и **TDMA** (временное разделение).

### Частотный план

Диапазон 900 MHz разбит на **каналы шириной 200 kHz**. Каждый канал идентифицируется номером **ARFCN** (Absolute Radio Frequency Channel Number - абсолютный номер радиочастотного канала):

$$f_{uplink}(n) = 890{,}2 + 0{,}2 \cdot (n-1) \text{ MHz}, \quad n = 1..124$$
$$f_{downlink}(n) = f_{uplink}(n) + 45 \text{ MHz}$$

Разнос 45 MHz между uplink (от телефона к базовой станции) и downlink (от базовой к телефону) нужен, чтобы приём и передача не мешали друг другу.

### Структура фрейма TDMA

Один 200 kHz-канал делится во времени на **8 тайм-слотов** (time slot). Набор из 8 тайм-слотов образует один **TDMA-фрейм** длительностью **4.615 мс**. Один тайм-слот = **0.577 мс** = **156.25 бит** при скорости 270.833 kbps.

```
TDMA фрейм (4.615 мс):
|  TS0  |  TS1  |  TS2  |  TS3  |  TS4  |  TS5  |  TS6  |  TS7  |
  0.577   0.577   0.577   0.577   0.577   0.577   0.577   0.577
```

Абонент занимает **один тайм-слот** в каждом фрейме -> 8 абонентов на один 200 kHz-канал. Это и есть TDMA-мультидоступ.

### Структура нормального бёрста (Normal Burst)

Каждый тайм-слот содержит **бёрст** (burst - короткий пакет данных). Нормальный бёрст:

```
| 3 хвостовых | 58 бит данных | 26 обучающих | 58 бит данных | 3 хвостовых | 8.25 гард |
```

- **Обучающая последовательность** (Training Sequence - заранее известный узор битов) - нужна приёмнику, чтобы оценить и скомпенсировать искажения канала.
- **Защитный интервал** (Guard Period - 8.25 бит) - буфер между бёрстами разных абонентов, компенсирует разницу в задержках распространения.

## Модуляция GMSK

GSM использует **GMSK** - Gaussian Minimum Shift Keying (гауссовская манипуляция с минимальным сдвигом). Это разновидность частотной манипуляции (FSK), где биты кодируются сдвигом частоты, а форма импульса сглаживается **гауссовским фильтром**.

Для MSK (Minimum Shift Keying) сдвиг частоты равен $\pm \Delta f$, причём:

$$\Delta f = \frac{R_b}{4}, \quad \text{где } R_b = 270{,}833 \text{ kbps} \quad \Rightarrow \quad \Delta f = 67{,}7 \text{ kHz}$$

Индекс модуляции: $h = \frac{2 \Delta f}{R_b} = 0{,}5$ (отсюда "минимальный" сдвиг).

Математически GMSK-сигнал:

$$s(t) = A_c \cos\!\Big(2\pi f_c t + \phi(t)\Big),$$

где фаза $\phi(t)$ меняется непрерывно. Биты $\{+1, -1\}$ прогоняются через **гауссовский фильтр** с нормированной полосой $BT = 0{,}3$ (B — полоса фильтра, T — длительность бита), что сглаживает мгновенные фазовые скачки и уменьшает занимаемую полосу. Цена — небольшая межсимвольная интерференция (ISI — InterSymbol Interference, «хвост» одного бита влияет на следующий).

## Канальное кодирование и перемежение

### Свёрточное кодирование (Convolutional Coding)

Биты речи защищаются свёрточным кодом. Для FR речи применяется код со скоростью $R = 1/2$ и длиной памяти $K = 5$: на каждый входной бит выходит 2 выходных, что удваивает избыточность но позволяет исправлять ошибки декодером Витерби (Viterbi decoder — алгоритм поиска наиболее вероятной последовательности битов через решётчатую диаграмму).

### Перемежение (Interleaving)

Радиоканал имеет **пакетные ошибки** (burst errors) — когда абонент находится в зоне глубокого замирания (фейдинга), несколько последовательных бит могут быть потеряны. Свёрточный код плохо справляется с пакетными ошибками.

Решение: **перемежение** — биты одного блока речи перемешиваются и раскладываются по нескольким бёрстам (в GSM — по 8 последовательным бёрстам). В итоге пакетная ошибка превращается в одиночные ошибки, рассредоточенные по разным блокам → свёрточный декодер легко их исправляет.

## Безопасность в GSM: SIM, аутентификация, шифрование

### SIM (Subscriber Identity Module)

SIM — смарт-карта, хранящая:
- **IMSI** (International Mobile Subscriber Identity — международный идентификатор абонента)
- **Ki** — секретный 128-битный ключ (никогда не покидает SIM и HLR)
- Алгоритмы A3 (аутентификация) и A8 (генерация ключа шифрования)

### Аутентификация (алгоритм A3)

Сеть отправляет телефону случайное число **RAND** (128 бит). SIM вычисляет:

$$SRES = A3(Ki,\, RAND) \quad \text{(Signed RESponse, 32 бита)}$$

Сеть независимо вычисляет то же значение по своей копии Ki и RAND. Если SRES совпал — абонент аутентифицирован. Это **challenge-response** аутентификация (протокол «вызов-ответ», при котором сторона доказывает знание секрета без его раскрытия).

### Генерация ключа шифрования (алгоритм A8)

Из того же RAND и Ki генерируется **сессионный ключ Kc** (64 бит):

$$Kc = A8(Ki,\, RAND)$$

### Шифрование (алгоритм A5)

Данные радиоканала шифруются алгоритмом **A5**. Версии:
- **A5/1** — сильный потоковый шифр на регистрах сдвига с линейной обратной связью (LFSR — Linear Feedback Shift Register), использовался в Европе.
- **A5/2** — намеренно ослабленный экспортный вариант (уже взломан).
- **A5/0** — отсутствие шифрования.

Ключевой поток генерируется из Kc и номера текущего фрейма TDMA. XOR ключевого потока с данными = зашифрованный поток.

> **Важный изъян GSM**: аутентификация **односторонняя** — телефон доказывает сети свою подлинность, но **сеть не доказывает телефону свою подлинность**. Это позволяет атаку через поддельную базовую станцию (IMSI Catcher / Stingray).

## Handover (переключение между базовыми станциями)

Когда абонент движется, качество сигнала текущей BTS падает. GSM выполняет **handover** — переключение на другую BTS без разрыва звонка.

Телефон непрерывно измеряет **RxLev** (уровень принятого сигнала, в dBm) и **RxQual** (качество, по BER — Bit Error Rate, доля ошибочных бит) текущей и соседних ячеек и отправляет эти данные в BSC. BSC принимает решение о handover.

Алгоритм основан на пороговых значениях:

$$RxLev_{neighbour} - RxLev_{serving} > HO\_MARGIN$$

Handover в GSM — **жёсткий** (hard handover): связь со старой BTS разрывается до установления связи с новой (разрыв порядка 100–200 мс). В CDMA реализован **мягкий хэндовер** (soft handover) — подробнее ниже.

## GPRS и EDGE: пакетные данные поверх GSM

### GPRS (General Packet Radio Service, «2.5G»)

GPRS добавляет в сеть **пакетные узлы**: SGSN (Serving GPRS Support Node) и GGSN (Gateway GPRS Support Node), которые обеспечивают IP-подключение. Несколько тайм-слотов одного TDMA-фрейма могут объединяться под один абонентский пакетный поток (до 8 слотов).

Схемы кодирования CS-1..CS-4 дают разные соотношения скорости и защиты:

| Схема | Скорость на слот | Кодирование |
|---|---|---|
| CS-1 | 9.05 kbps | сильная защита |
| CS-2 | 13.4 kbps | |
| CS-3 | 15.6 kbps | |
| CS-4 | 21.4 kbps | нет FEC |

Максимум при 8 слотах CS-4: $\approx 171$ kbps (теоретически), реально 20–40 kbps.

### EDGE (Enhanced Data rates for GSM Evolution, «2.75G»)

EDGE меняет **модуляцию**: вместо GMSK применяется **8-PSK** (Phase Shift Keying — фазовая манипуляция с 8 уровнями), что позволяет кодировать 3 бита на символ вместо 1:

$$\log_2(8) = 3 \text{ бит/символ}$$

Новые схемы кодирования MCS-1..MCS-9 дают до **59.2 kbps на тайм-слот**, и до ~384 kbps при 8 слотах. 8-PSK требует лучшего SNR, поэтому система адаптивно выбирает схему в зависимости от качества канала.

## IS-95 / cdmaOne: принцип CDMA

В IS-95 все абоненты передают **одновременно в одной полосе 1.25 MHz**. Разделение достигается за счёт **расширения спектра псевдошумовой последовательностью** (Spread Spectrum — уширение спектра до полосы много шире исходного сигнала).

### Расширение спектра (Direct Sequence Spread Spectrum, DSSS)

Пусть речевой сигнал после CODEC имеет скорость $R_b = 9{,}6$ kbps, а PN-последовательность (Pseudo-Noise — псевдошумовая, выглядит как случайная, но детерминирована начальным состоянием) имеет скорость **чипов** $R_c = 1{,}2288$ Mcps (миллионов chips per second). Коэффициент расширения (Processing Gain):

$$PG = \frac{R_c}{R_b} = \frac{1{,}2288 \times 10^6}{9600} = 128$$

В децибелах: $PG_{dB} = 10 \log_{10}(128) \approx 21$ dB.

Передатчик: бит данных XOR с PN-последовательностью → широкополосный сигнал. Приёмник: такая же PN-последовательность XOR с принятым сигналом → исходные данные. Сигналы других абонентов с разными PN-кодами выглядят как некоррелированный шум.

### Ортогональные коды Уолша (Walsh Codes)

IS-95 downlink использует **64 ортогональных кода Уолша** длиной 64 чипа для разделения каналов. Ортогональность означает:

$$\sum_{i=0}^{63} w_k[i] \cdot w_m[i] = 0, \quad k \neq m$$

Для разных абонентов скалярное произведение равно нулю → нет взаимных помех (в идеальном канале).

### Управление мощностью (Power Control)

Ключевая проблема CDMA — **near-far problem** (проблема «ближний-дальний»): если один абонент ближе к базовой, его сигнал может «забить» сигналы дальних абонентов, так как мощность при распространении убывает как $\sim d^{-n}$. Решение: **быстрое управление мощностью**.

- **Разомкнутая петля** (Open Loop): телефон измеряет мощность принятого downlink-сигнала и обратно пропорционально устанавливает свою мощность uplink.
- **Замкнутая петля** (Closed Loop, 800 Гц): базовая станция каждые 1.25 мс посылает команду «увеличить/уменьшить мощность на 1 dB». Телефон немедленно подчиняется.

Цель: все абоненты принимаются базовой станцией с **одинаковой мощностью** (± несколько дБ).

### Мягкий хэндовер (Soft Handover)

В CDMA все соты используют **одну и ту же частоту**. При перемещении абонента в зону перекрытия двух сот телефон одновременно поддерживает связь с **несколькими** базовыми станциями. Обе принимают сигнал и передают его в MSC, где применяется алгоритм **макроразнесения** (Macro Diversity — объединение сигналов от нескольких базовых станций) — выбирается или комбинируется лучший. Разрыва соединения нет — отсюда название «мягкий».

## Распространение сигнала и модели потерь

### Логарифмическая модель потерь (Log-Distance Path Loss)

Реальные условия городской застройки описываются обобщённой моделью:

$$PL(d) = PL(d_0) + 10 \cdot n \cdot \log_{10}\!\left(\frac{d}{d_0}\right) + X_\sigma,$$

где:
- $d_0$ — опорное расстояние (обычно 1 м или 1 км),
- $n$ — показатель потерь (в свободном пространстве $n=2$; в городе $n=3{,}5$–$4{,}5$),
- $X_\sigma$ — случайная величина с нормальным распределением $\mathcal{N}(0,\sigma^2)$, теневые замирания (shadowing — медленные флуктуации из-за зданий и рельефа).

### Модель Хата (Okumura-Hata)

Эмпирическая модель для городской/пригородной застройки в диапазоне 150–1500 MHz:

$$PL_{urban}(\text{дБ}) = 69{,}55 + 26{,}16 \log f_c - 13{,}82 \log h_b - a(h_m) + (44{,}9 - 6{,}55 \log h_b)\log d,$$

где $f_c$ — несущая [MHz], $h_b$ — высота базовой антенны [м], $h_m$ — высота мобильной антенны [м], $d$ — расстояние [км], $a(h_m)$ — поправочный коэффициент на высоту мобильной антенны.

## Бюджет линии (Link Budget)

Бюджет линии — расчёт, позволяющий определить максимально допустимые потери пути $PL_{max}$:

$$EIRP = P_t [\text{dBm}] + G_t [\text{dBi}] - L_{cable} [\text{dB}]$$

$$P_{rx,min} = S_{rx} [\text{dBm}] - G_r [\text{dBi}] + L_{body} [\text{dB}]$$

$$PL_{max} = EIRP - P_{rx,min}$$

где $EIRP$ — эффективная изотропно-излучаемая мощность (Effective Isotropic Radiated Power), $S_{rx}$ — чувствительность приёмника, $G$ — усиление антенны, $L$ — потери.

## Отношение сигнал/помеха в GSM: C/I

Для нормальной работы GSM необходимо, чтобы отношение несущей к помехе $C/I$ превышало порог:

$$\frac{C}{I} \geq 9 \text{ dB}$$

Это ограничивает **коэффициент переиспользования частот** $N$ (число ячеек в кластере, после которого частоты повторяются). При гексагональной модели:

$$D/R = \sqrt{3N}, \quad \text{где } D - \text{расстояние до первой сосотрудничающей соты, } R - \text{радиус соты.}$$

Типичные кластеры: $N = 4$ (при секторизации) или $N = 7$ для омни-антенн.

## Уровни стека протоколов GSM (Um интерфейс)

```
   MS                BTS                BSC/MSC
┌──────┐          ┌──────┐            ┌──────┐
│  CM  │◄────────────────────────────►│  CM  │  Call Management
│  MM  │◄────────────────────────────►│  MM  │  Mobility Management
│  RR  │◄─────────►│  RR  │◄─────────►│  RR  │  Radio Resource
│ LAPDm│◄─────────►│LAPDm │                    Data Link (HDLC-подобный)
│  PL  │◄─────────►│  PL  │                    Physical Layer
└──────┘          └──────┘
```

- **PL** (Physical Layer) — бёрсты, TDMA, GMSK.
- **LAPDm** — канальный уровень, обеспечивает надёжную доставку сигнальных сообщений.
- **RR** (Radio Resource) — управление радиоресурсами: назначение каналов, хэндовер, измерения.
- **MM** (Mobility Management) — регистрация, аутентификация, обновление местоположения (Location Update).
- **CM** (Connection Management) — управление вызовами, SMS, дополнительные услуги.

## Ёмкость сети

Ёмкость GSM-системы ограничена числом доступных тайм-слотов. Трафик одного канала (1 тайм-слот) принято описывать **Erlang-моделью** (Эрланг — единица телефонной нагрузки). Пропускная способность соты из $C$ каналов при допустимой вероятности блокировки $B$ определяется **формулой Erlang-B**:

$$B = \frac{A^C / C!}{\sum_{k=0}^{C} A^k / k!},$$

где $A$ — суммарная трафиковая нагрузка в Эрланг. Задача планирования: выбрать $C$ таким, чтобы $B \leq 2\%$.

## Ограничения 2G

- Скорость данных невысока даже с EDGE (до 384 kbps — «3G-lite»).
- Аутентификация односторонняя → уязвимость к IMSI Catcher.
- Шифрование A5/1 теоретически взломано при наличии радужных таблиц.
- Голос передаётся через CS-домен (Circuit Switched — коммутация каналов), а не через IP → нет VoIP по умолчанию.
