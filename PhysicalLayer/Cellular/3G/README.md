# 3G - UMTS / WCDMA / HSPA

3G - третье поколение сотовых сетей, стандартизированное консорциумом **3GPP** (3rd Generation Partnership Project). Основной стандарт - **UMTS** (Universal Mobile Telecommunications System), радиоинтерфейс - **WCDMA** (Wideband CDMA). Ключевые отличия от 2G: более высокая скорость передачи данных (до 2 Mbps при первоначальном стандарте), одновременная передача голоса и данных (CS + PS домены), значительно более широкая полоса радиоканала (5 MHz против 200 kHz GSM). Позднее UMTS эволюционировал в **HSPA** (High Speed Packet Access), увеличив скорости до десятков Mbit/s.

---

## Краткий обзор

| Параметр | UMTS R99 | HSDPA (3.5G) | HSUPA | HSPA+ |
|---|---|---|---|---|
| Полоса канала | 5 MHz | 5 MHz | 5 MHz | 5 MHz |
| Скорость DL | до 384 kbps | до 14.4 Mbps | - | до 42 Mbps |
| Скорость UL | до 384 kbps | - | до 5.76 Mbps | до 11.5 Mbps |
| Модуляция DL | QPSK | QPSK / 16-QAM | QPSK | 64-QAM |
| Мультидоступ | WCDMA | WCDMA | WCDMA | WCDMA |
| Частоты | 2100 MHz (основной) | - | - | - |
| Задержка RTT | ~150 мс | ~70 мс | ~30 мс | ~10 мс |

## Архитектура сети UMTS

```
UE (телефон)
    |
  Uu (радиоинтерфейс WCDMA)
    |
  Node B (аналог BTS в GSM)
    |
  Iub
    |
  RNC (Radio Network Controller - контроллер радиосети)
    |    \
   Iur   Iu-CS          Iu-PS
    |       |               |
  другой   MSC/VLR       SGSN
   RNC      |               |
            PSTN           GGSN → Internet
```

- **Node B** - базовая станция, выполняет физическую обработку сигнала: модуляцию/демодуляцию, расширение спектра, RAKE-приём. В отличие от GSM BTS имеет более сложную обработку.
- **RNC** (Radio Network Controller) - управляет несколькими Node B. Выполняет: управление радиоресурсами (RRM), хэндовер, управление мощностью (в отличие от GSM, где это делал BSC), сборку/разборку протоколов RLC/MAC.
- **Iu-CS** - интерфейс к домену коммутации каналов (голос).
- **Iu-PS** - интерфейс к домену пакетной коммутации (данные, IP).
- **SGSN** (Serving GPRS Support Node) - маршрутизирует пакеты к UE, ведёт учёт сессий.
- **GGSN** (Gateway GPRS Support Node) - шлюз к внешним IP-сетям (интернет).

## WCDMA: принцип работы

### Широкополосное расширение спектра

WCDMA использует **прямое расширение спектра** (DS-SS - Direct Sequence Spread Spectrum). Данные умножаются на **чиповую последовательность** со скоростью $R_c = 3{,}84$ Mchips/s (мегачипов в секунду). Полоса канала $\approx 5$ MHz.

Коэффициент расширения (Spreading Factor, SF) для данных со скоростью $R_b$:

$$SF = \frac{R_c}{R_b} = \frac{3{,}84 \times 10^6}{R_b}$$

Примеры:
- Речь 12.2 kbps: $SF = 3840/12.2 \approx 256$
- Данные 384 kbps: $SF = 3840/384 = 10$
- Данные 2 Mbps: $SF = 3840/2000 \approx 2$ (минимальный для UMTS R99 - реально SF=4)

### Двухуровневая структура кодирования

WCDMA использует **два вида кодов** одновременно:

1. **Коды канализации** (Channelization Codes, OVSF - Orthogonal Variable Spreading Factor): ортогональные коды Уолша переменной длины. Разделяют каналы одного абонента или разных абонентов **внутри одной соты**. Ортогональность обеспечивает нулевые взаимные помехи:
   $$\langle c_k, c_m \rangle = \sum_{i} c_k[i] \cdot c_m[i] = 0, \quad k \neq m$$

2. **Скремблирующие коды** (Scrambling Codes): длинные Gold-коды (псевдослучайные последовательности, построенные на двух максимально длинных сдвиговых регистрах). Разделяют **разные соты**. Длина: 38400 чипов = 10 мс. В downlink каждая Node B использует уникальный скремблирующий код -> абонент идентифицирует соту по этому коду.

Процесс передачи: данные -> OVSF-код -> скремблирующий код -> RF передача.

### RAKE-приёмник

В широкополосном канале одна и та же ПН-последовательность приходит с **разными задержками** по разным путям (многолучевость). Поскольку $R_c = 3{,}84$ Mchips/s, разрешение по задержке = $1/R_c \approx 0{,}26$ мкс ≈ **78 м по пути**. Пути, отличающиеся более чем на один чип, можно **разрешить и объединить**.

**RAKE-приёмник** - набор независимых коррелятров (fingers - "пальцев"), каждый настроен на свою задержку. Сигнал каждого finger коррелируется с PN-кодом при данной задержке -> получает детерминированный сигнал данного пути. Затем все fingers объединяются с весами, пропорциональными SNR каждого пути (Maximum Ratio Combining, MRC):

$$\hat{s} = \sum_{l=1}^{L} \alpha_l^* \cdot r_l,$$

где $\alpha_l$ - комплексный коэффициент замирания $l$-го пути, $r_l$ - принятый символ в $l$-м finger.

Это превращает многолучевость из врага в **союзника** - дополнительное разнесение сигнала.

## Управление мощностью в WCDMA

Управление мощностью критически важно, т.к. все абоненты делят одну полосу и мешают друг другу. WCDMA реализует **трёхуровневое** управление:

### Разомкнутая петля (Open-Loop PC)

При начальном доступе к сети телефон оценивает потери пути по downlink-пилоту и устанавливает начальную мощность UL:

$$P_{TX} = -P_{RX} + C_{constant}$$

### Замкнутая внутренняя петля (Inner-Loop PC, 1500 Гц)

Node B каждые **0.667 мс** (каждый слот) измеряет SIR (Signal-to-Interference Ratio - отношение сигнала к помехе) принятого сигнала от UE и сравнивает с целевым значением $SIR_{target}$:

$$TPC\_command = \begin{cases} +1 \text{ dB} & \text{если } SIR < SIR_{target} \\ -1 \text{ dB} & \text{если } SIR \geq SIR_{target} \end{cases}$$

Команда TPC (Transmit Power Control) передаётся в UE. Скорость реакции - 1500 раз/с, что позволяет отслеживать **быстрый фейдинг** (скорость изменения канала при скорости 50 км/ч на 2.1 GHz ≈ 100 Гц).

### Внешняя петля (Outer-Loop PC)

RNC регулирует $SIR_{target}$ для поддержания целевого BLER (Block Error Rate - вероятность ошибочного блока), обычно 10% для речи, 1% для данных. Если BLER растёт -> RNC повышает $SIR_{target}$.

## Голос в UMTS: AMR CODEC

UMTS использует **AMR** (Adaptive Multi-Rate) кодек. AMR переключается между 8 режимами: 4.75, 5.15, 5.9, 6.7, 7.4, 7.95, 10.2, 12.2 kbps. В плохих условиях канала снижается скорость -> больше защитного кодирования -> качество речи лучше; в хороших условиях -> выше скорость -> натуральнее речь.

Канальное кодирование для речи 12.2 kbps:
- 81 бит класса A (наиболее чувствительные - параметры LPC) -> защищены CRC(8) + свёрточный R=1/3
- 103 бита класса B -> свёрточный R=1/2
- 60 бит класса C -> без кодирования

После кодирования - **перемежение** по 20 мс блоку.

## Структура физического уровня WCDMA

### Временна́я структура (Frame/Slot Structure)

```
Суперфрейм: 720 мс = 72 радиофрейма
Радиофрейм: 10 мс = 15 слотов
Слот: 10/15 мс ≈ 0.667 мс = 2560 чипов
```

Скорость: $15 \times 2560 = 38400$ чипов/фрейм = $38400 \times 100 = 3{,}84$ Mchips/s ✓

### Физические каналы (Downlink)

- **CPICH** (Common Pilot Channel) - общий пилотный канал, SF=256, мощность ~10% от общей. Используется UE для оценки канала, измерений соседних сот, синхронизации.
- **P-CCPCH** (Primary Common Control Physical Channel) - передаёт BCH (Broadcast Channel) с системной информацией.
- **DPCH** (Dedicated Physical Channel) - индивидуальный канал абонента. Содержит DPDCH (данные) + DPCCH (управление: пилот, TFCI, TPC, FBI).

### Структура слота DPCH (downlink)

```
| DPDCH (данные 1) | DPCCH (пилот+TFCI+TPC) | DPDCH (данные 2) |
```

- **TFCI** (Transport Format Combination Indicator) - сообщает приёмнику, с какой скоростью/кодированием переданы данные в текущем фрейме.
- **TPC** - команды управления мощностью.
- **Pilot bits** - обучающие биты для оценки канала.

## HSDPA — ускорение нисходящего канала

**HSDPA** (High Speed Downlink Packet Access, 3GPP Release 5) — надстройка над UMTS, которая переносит часть логики из RNC в Node B и добавляет новые механизмы:

### Адаптивная модуляция и кодирование (AMC)

В обычном UMTS скорость поддерживается управлением мощностью. HSDPA вместо этого **фиксирует мощность** и **меняет модуляцию/кодирование** в зависимости от качества канала (CQI — Channel Quality Indicator, который UE отправляет каждые 2 мс):

| CQI | Модуляция | Кодирование | Скорость |
|---|---|---|---|
| низкий | QPSK | R = 1/4 | ~1 Mbps |
| средний | 16-QAM | R = 1/2 | ~5 Mbps |
| высокий | 16-QAM | R = 3/4 | ~14.4 Mbps |

16-QAM (Quadrature Amplitude Modulation — квадратурная амплитудная модуляция) кодирует 4 бита на символ против 2 бит для QPSK.

### HARQ (Hybrid ARQ)

**HARQ** (Hybrid Automatic Repeat reQuest — гибридный запрос повторной передачи) объединяет FEC (Forward Error Correction — упреждающее исправление ошибок) и ARQ (автоматический запрос повторной передачи):

- Если блок принят с ошибками → запрашивается ретрансмиссия.
- Ретрансмиссия **комбинируется** с первой передачей в приёмнике (Chase Combining или Incremental Redundancy).
- Incremential Redundancy: каждая ретрансмиссия содержит **дополнительные биты избыточности**, а не повтор тех же данных. В итоге накапливаемая информация улучшает декодирование.

**Chase Combining**: принятые биты складываются как векторы → SNR растёт как $\sim N$ (N — число попыток).

### Планировщик Node B (Scheduling)

Ключевое нововведение: планировщик ресурсов перенесён из RNC в Node B. Решение о том, кому дать ресурс в следующем TTI (Transmission Time Interval = 2 мс), принимается за **2 мс**. Типичная стратегия — **Proportional Fair**: выбирается UE с максимальным отношением $R_k / \bar{R}_k$, где $R_k$ — мгновенная достижимая скорость, $\bar{R}_k$ — средняя скорость по истории.

## HSUPA — ускорение восходящего канала

**HSUPA** (High Speed Uplink Packet Access, 3GPP Release 6) — аналог для uplink. Вводит:
- **E-DCH** (Enhanced Dedicated Channel): TTI = 2 мс или 10 мс.
- HARQ с Incremental Redundancy.
- Планировщик Node B для UL: UE запрашивает ресурс через «scheduling grant», Node B выдаёт разрешение.
- Максимальная скорость: **5.76 Mbps**.

---

## HSPA+: дальнейшая эволюция

**HSPA+** (Release 7, 8) добавляет:
- **64-QAM** в downlink: 6 бит/символ → пиковая скорость до **42 Mbps**.
- **16-QAM** в uplink → до **11.5 Mbps**.
- **2×2 MIMO** (Multiple-Input Multiple-Output — несколько антенн на передаче и приёме) в downlink: два независимых потока данных = ×2 к скорости.
- Flat RAN архитектура: возможность прямого соединения Node B–Node B (Iur-интерфейс).

## Мягкий хэндовер (Soft Handover)

В WCDMA все соты работают на **одной частоте** (частоты переиспользуются в каждой соте). При перемещении абонента в зону перекрытия двух сот устанавливается **Active Set** — множество Node B, с которыми UE одновременно поддерживает связь.

- **Downlink**: UE принимает сигнал от нескольких Node B, RAKE-приёмник объединяет.
- **Uplink**: несколько Node B принимают сигнал от UE, RNC выбирает лучший блок (Selection Diversity).

Преимущество: нет потери связи при переходе (в отличие от жёсткого хэндовера GSM). Недостаток: UE тратит мощность на поддержание нескольких линий.

Критерий входа в Active Set: сигнал соты превышает $P_{best} - AS\_THRESHOLD$ (в дБ). Типично $AS\_THRESHOLD = 3$–$6$ дБ.

## Ёмкость WCDMA и дыхание соты (Cell Breathing)

В WCDMA добавление каждого нового абонента **повышает уровень интерференции** для всех остальных. Суммарная мощность помех (Interference Rise):

$$\eta = \frac{I_{total}}{I_{other}} = \frac{1}{1 - \rho},$$

где $\rho$ — загрузка соты (Uplink Load Factor). При $\rho \to 1$ → $\eta \to \infty$ → сота «перегружена».

Практический порог: $\rho_{UL} \leq 0{,}5$–$0{,}75$.

**Cell Breathing** (дыхание соты): с ростом числа абонентов потребная мощность растёт, зона покрытия **сжимается** (т.к. тот же порог SINR требует большей мощности передачи на малом расстоянии). Сети 3G при высокой нагрузке физически уменьшают покрытие каждой соты.

Пропускная способность соты WCDMA (Uplink):

$$N_{max} = \frac{(1-\eta_{target})}{\frac{1}{SF \cdot (E_b/N_0)_{req}} + \frac{\alpha}{N}},$$

где $E_b/N_0$ — отношение энергии на бит к спектральной плотности шума (мера качества принятого сигнала), $\alpha$ — соотношение внутрисотовой и внешней интерференции.

## Модели канала в UMTS

3GPP определяет стандартные модели канала для тестирования:
- **AWGN** — только аддитивный белый гауссов шум (модель без многолучевости).
- **Pedestrian A/B** — пешеходная среда: 2–6 лучей, задержки от 0 до 410 нс.
- **Vehicular A/B** — автомобильная среда: большие задержки до 2510 нс, сильный доплер.

Доплеровский сдвиг при скорости $v$:

$$f_D = \frac{v \cdot f_c}{c} \cos\theta,$$

где $c = 3 \times 10^8$ м/с. При $v = 120$ км/ч, $f_c = 2100$ MHz: $f_D^{max} = \frac{(120/3.6) \times 2100 \times 10^6}{3 \times 10^8} \approx 233$ Гц.

## Стек протоколов UMTS (Uu интерфейс)

```
┌─────────────────────────────────────┐
│     CM / MM / SM (NAS уровень)      │  Non-Access Stratum — конец-в-конец между UE и CN
├─────────────────────────────────────┤
│              RRC                    │  Radio Resource Control — управление радиоресурсами
├─────────────────────────────────────┤
│         PDCP (только PS)            │  Packet Data Convergence Protocol — заголовочное сжатие
├─────────────────────────────────────┤
│              RLC                    │  Radio Link Control — сегментация, ARQ
├─────────────────────────────────────┤
│              MAC                    │  Medium Access Control — мультиплексирование, HARQ
├─────────────────────────────────────┤
│              PHY                    │  Physical Layer — WCDMA, управление мощностью
└─────────────────────────────────────┘
```

- **RRC** — главный управляющий: устанавливает/освобождает радиосоединения, конфигурирует PHY/MAC/RLC, управляет измерениями для хэндовера.
- **RLC** — три режима: TM (Transparent Mode), UM (Unacknowledged Mode), AM (Acknowledged Mode с ARQ).
- **PDCP** — сжатие заголовков IP (ROHC — Robust Header Compression), опциональное шифрование.

## Безопасность UMTS: устранение уязвимостей GSM

UMTS исправляет ключевую уязвимость GSM — добавляет **взаимную аутентификацию**:

1. Сеть генерирует RAND и **AUTN** (Authentication Token — токен, вычисленный из Ki, RAND и счётчика SQN).
2. UE проверяет AUTN → убеждается, что это **настоящая** сеть, а не фальшивая BTS.
3. UE вычисляет RES (Response), сеть верифицирует.

Кроме того, шифрование применяется уже на уровне **RLC** (не только на физическом, как в GSM), а также добавляется **защита целостности** (Integrity Protection) для RRC-сообщений с кодом MAC-I.

---

## Ограничения 3G

- Задержка RTT в UMTS R99: 100–150 мс — слишком велика для чувствительных приложений.
- Сложность сети (RNC — узкое место).
- Эффект cell breathing при высокой нагрузке.
- Скорость на краю соты значительно ниже пиковой.
- Не полностью IP-архитектура (CS-домен для голоса сохранён).
