# HTTP/1.1

HTTP (HyperText Transfer Protocol) - это протокол прикладного уровня, который используется для передачи данных между клиентом (например, браузером) и сервером. Он является основой для обмена информацией в интернете. HTTP работает поверх TCP/IP и использует стандартные порты: 80 для HTTP и 443 для HTTPS.

## Теория

Основные характеристики HTTP/1.1:
- **Статeless**: HTTP не сохраняет состояние между запросами. Каждый запрос обрабатывается независимо от предыдущих
- **Методы**: HTTP поддерживает различные методы, такие как GET, POST, PUT, DELETE и другие, которые определяют действие, которое клиент хочет выполнить с ресурсом
- **Заголовки**: HTTP использует заголовки для передачи метаданных о запросе или ответе, таких как тип контента, длина тела и другие
- **Коды состояния**: Сервер возвращает коды состояния, чтобы указать результат обработки запроса (например, 200 OK, 404 Not Found, 500 Internal Server Error)
- **Постоянные соединения**: HTTP/1.1 поддерживает постоянные соединения, что позволяет использовать одно TCP-соединение для нескольких запросов и ответов, улучшая производительность
- **Кэширование**: HTTP/1.1 включает механизмы кэширования, которые позволяют клиентам и серверам сохранять копии ресурсов для уменьшения задержек и нагрузки на сервер
- **Поддержка виртуальных хостов**: HTTP/1.1 позволяет одному серверу обслуживать несколько доменов, используя заголовок Host для определения целевого ресурса

Рассмотрим структуру HTTP-запроса:
```c
enum http_method {
    GET,
    POST,
    PUT,
    DELETE,
    UNKNOWN
};

struct http_request {
    enum http_method method;
    char *url;
    char *version;
    char *hearders;
    char *body;
};
```

1. Стартовая строка :

    - Метод (GET, POST, PUT, DELETE и так далее)

    - URL (путь к ресурсу)

    - Версия протокола (например, HTTP/1.1)

2. Заголовки :

    - Дополнительная информация о запросе (например, тип контента, авторизация, язык клиента)

3. Тело:

    - Данные, которые отправляются в запросе (например, данные формы или JSON)
