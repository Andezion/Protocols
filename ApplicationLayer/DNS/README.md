DNS (Domain Name System)

DNS это система, которая переводит понятные человеку доменные имена в IP-адреса, которые используют компьютеры для связи друг с другом. Представьте, что DNS – это как телефонная книга интернета. В обычной телефонной книге, если вам нужно найти номер телефона человека, вы ищете его по имени, и книга предоставляет вам номер. Точно так же DNS помогает находить IP-адреса, когда вы вводите доменное имя.

## Теория 

Когда вы вводите адрес веб-сайта, например, www.example.com, в строку браузера, DNS помогает найти нужный IP-адрес, чтобы ваш браузер мог подключиться к нужному серверу. IP-адрес – это уникальный числовой идентификатор, который используется для идентификации устройства в сети. Всякий раз, когда вы отправляете запрос на веб-сайт, ваш браузер отправляет этот запрос через интернет на сервер, который затем отвечает данными, которые вы видите на экране.

Процесс начинается с того, что ваш компьютер или устройство отправляет запрос на ближайший DNS-сервер, который обычно предоставляется вашим интернет-провайдером. Этот сервер называется рекурсивным резолвером. Если этот сервер знает нужный IP-адрес, он возвращает его вашему браузеру. Если нет, запрос передается на другие DNS-серверы, пока не будет найден правильный адрес. Этот процесс может включать несколько шагов, таких как обращение к корневым серверам, серверам доменных зон и авторитетным серверам, которые хранят информацию о конкретных доменах. В итоге, когда нужный IP-адрес найден, он возвращается вашему браузеру, который затем устанавливает соединение с сервером веб-сайта и загружает страницу.

## Практика

Когда вы вводите адрес сайта в строку браузера, происходит следующий процесс:

1) Проверка кэша: Устройство сначала ищет нужный IP-адрес в памяти самого компьютера или в роутере.
2) Запрос к провайдеру: Если данных нет, запрос отправляется DNS-серверу вашего интернет-провайдера (резолверу).
3) Иерархия серверов: Если провайдер не знает адрес, запрос проходит по цепочке:
    - Корневые серверы (Root servers): Направляют на серверы доменных зон.
    - Серверы доменных зон (TLD-серверы): Отвечают за зоны вроде .com, .net, .org и знают, где искать сайт.
    - Авторитетные серверы (Authoritative name servers): Предоставляют точный IP-адрес конкретного сайта.
4) Ответ пользователю: IP-адрес возвращается на ваше устройство, браузер устанавливает связь с сервером сайта, и страница загружается. Этот процесс занимает доли секунды
5) Сохранение (Кэширование): Чтобы не проделывать этот путь каждый раз, найденный IP-адрес временно сохраняется в кэше устройства и провайдера, что ускоряет последующие запросы к тому же сайту.

Допустим мы ищем какой-то example.com:

1) Локальный кеш: сначала браузер проверяет свой локальный кеш на наличие записи для example.com. Если она найдена, используется закешированный IP-адрес.

2) Рекурсивный резолвер: если записи нет в локальном кеше, запрос отправляется на рекурсивный DNS-сервер. Этот сервер выполняет роль посредника, который ищет нужную информацию, обращаясь к другим серверам.

3) Корневые серверы: если рекурсивный сервер не знает IP-адрес, он отправляет запрос на один из корневых серверов. Корневые серверы знают, какой сервер отвечает за каждый домен верхнего уровня (TLD), такой как .com, .net, .org и так далее.

4) Сервер TLD: корневой сервер отвечает, какой сервер управляет доменом .com, и рекурсивный резолвер отправляет запрос на этот сервер.

5) Авторитетный сервер: сервер TLD (например, для .com) отвечает, какой сервер управляет доменом example.com. Рекурсивный резолвер направляет запрос на авторитетный DNS-сервер для example.com, который содержит точные данные о домене.

6) Возвращение IP-адреса: авторитетный сервер возвращает IP-адрес, связанный с example.com, рекурсивному резолверу, который, в свою очередь, возвращает его вашему браузеру.

7) Подключение к веб-сайту: теперь, когда браузер знает IP-адрес, он может отправить запрос непосредственно на сервер, чтобы загрузить содержимое веб-сайта.
